# Contributing to SCC Inspector

Thank you for your interest in contributing to SCC Inspector!

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Run tests to ensure nothing breaks (`python tests\test_all.py`)
5. Commit your changes (`git commit -m 'Add some feature'`)
6. Push to the branch (`git push origin feature/your-feature`)
7. Open a Pull Request

## Code Style

- Follow PEP 8 guidelines
- Add docstrings to new functions
- Keep functions focused and concise

## Testing

- Add tests for new features
- Ensure all existing tests pass before submitting PR
- Test with actual SCC files when possible

## Reporting Bugs

Open an issue with:
- Description of the bug
- Steps to reproduce
- Expected vs actual behavior
- Sample SCC file (if applicable)

## Feature Requests

Open an issue describing:
- The feature you'd like to see
- Why it would be useful
- Possible implementation approach
